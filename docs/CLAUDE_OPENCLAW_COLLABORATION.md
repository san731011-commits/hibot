# Claude Code + OpenClaw 협업 가이드

## 시나리오 1: Claude Code (Windows) + OpenClaw (라즈베리파이)

### 구조
```
[Windows PC - Claude Code]
        ↓ SSH/Tailscale
[Raspberry Pi - OpenClaw Agent]
        ↓ Discord/Telegram
[사용자]
```

### 장점
- 클로드는 윈도우에서 강력한 로컬 작업 가능
- OpenClaw는 24/7 라즈베리파이에서 실행
- 서로 SSH로 명령 전달 가능

### 구현 방법
1. 라즈베리파이에 SSH 키 설정
2. 클로드에서 `ssh pi@100.69.104.47`로 명령 실행
3. OpenClaw가 결과를 Discord/Telegram으로 전달

## 시나리오 4: 역할 분담

### Claude Code (Windows)
- **강점**: IDE 통합, 빠른 처리, 대용량 파일
- **담당**: 
  - 코드 작성/리팩토링
  - 복잡한 분석
  - 로컬 빌드/테스트
  - 문서 작성

### OpenClaw (라즈베리파이)
- **강점**: 24/7 실행, 메시지 알림, 자동화
- **담당**:
  - 모니터링/알림
  - Git 자동 커밋
  - 스케줄 작업
  - 메신저 연동

### 협업 흐름
```
1. 클로드가 코드 작성 (윈도우)
2. 클로드: "OpenClaw에게 푸시 요청"
3. OpenClaw: Git 푸시 실행 + 완료 알림
4. OpenClaw: Discord/Telegram으로 결과 전송
```

---

## 시나리오 5: Claude가 명령 생성, OpenClaw가 실행

### 예시
**클로드 (윈도우)**: 복잡한 배포 스크립트 작성
**OpenClaw (라즈베리파이)**: 실제 서버에서 실행 + 결과 모니터링

### 구현
```bash
# 클로드가 생성한 스크립트
#!/bin/bash
# deploy.sh - generated by Claude
npm run build
npm test
npm run deploy

# OpenClaw가 실행
curl -X POST http://localhost:18789/exec \
  --data @deploy.sh
```

## 추천 조합: 시나리오 1 + 4

### 가장 효율적인 구조
```
Claude Code (Windows)
├── 로컬 개발/분석 (강점 활용)
├── SSH로 라즈베리파이 명령 전달
└── 스크립트/설정 파일 생성

        ↓ SSH/Tailscale

OpenClaw (라즈베리파이)  
├── 24/7 실행
├── 자동화 작업
├── Git/시스템 작업 실행
└── Discord/Telegram 알림
```

### 실제 사용 예시
1. **코드 리뷰**: 클로드가 윈도우에서 분석 → OpenClaw가 PR 코멘트 작성
2. **배포**: 클로드가 빌드 → OpenClaw가 서버에 배포 + 알림
3. **모니터링**: OpenClaw가 서버 상태 체크 → 이상 시 클로드에게 알림 요청
4. **문서화**: 클로드가 문서 작성 → OpenClaw가 자동 푸시
